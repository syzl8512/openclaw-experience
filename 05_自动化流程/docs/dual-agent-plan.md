# 🤖 双机器人协作规划

> 创建: 2026-02-25

## 角色定义

| 机器人 | 位置 | 代号 | 主要职能 |
|--------|------|------|----------|
| **mini** | MacMini (本地) | main | 对话入口、调度中心 |
| **大龙虾** | 群晖 (syzl8512@ximi.space:2211) | node-1 | 文件处理、NAS任务、定时任务 |

---

## 📡 沟通机制

### 1. 消息传递
- 使用 `sessions_send` 跨机器人发消息
- 大龙虾作为 node 可被 nodes 工具调用

### 2. 触发方式
```
用户: 帮我下载xxx到群晖
→ mini 调度 → 大龙虾执行 → 返回结果
```

---

## 👁️ 监督机制

### 1. 状态检查
- 大龙虾运行状态监控
- 定时任务执行结果汇报

### 2. 日志同步
- 大龙虾的运行日志可被 mini 查看
- 重要事件推送到飞书/微信

---

## 🔧 服务机制

### 1. 任务类型

| 任务类型 | 执行位置 | 示例 |
|----------|----------|------|
| 对话响应 | mini | 直接回答 |
| 文件处理 | 大龙虾 | PDF转Word、批量重命名 |
| 下载任务 | 大龙虾 | 远程下载、BT下载 |
| 定时任务 | 大龙虾 | 每日备份、监控推送 |
| 家庭服务 | 大龙虾 | 语音助手、控制智能设备 |

### 2. 工作流示例

```
场景: 用户要求生成PPT
1. mini 接收请求
2. mini 调用 tavily-search 搜集素材
3. mini 调用 ppt-generator 生成
4. 如需大文件处理 → 转发大龙虾

场景: 用户需要下载视频
1. mini 接收URL
2. 转发大龙虾下载
3. 大龙虾完成 → 通知用户
```
