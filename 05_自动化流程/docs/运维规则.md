# 运维安全规则

## 1. 敏感信息管理

### 禁止上传到 GitHub 的内容

- `skills/server-manager/` - 包含服务器地址
- `config/system-info.md` - 包含服务器配置
- 任何含密码/密钥的配置文件

### 解决方案

1. **环境变量**：密码存储在 `~/.bashrc` 或环境变量中
2. **SSH密钥认证**：优先使用密钥连接
3. **.gitignore**：已在工作区添加 `.gitignore` 排除敏感配置

## 2. 密码存储规范

| 类型 | 存储位置 | 示例 |
|------|----------|------|
| SSH密钥 | ~/.ssh/ | id_rsa_maintenance |
| 密码 | 环境变量 | XIAOAI_PASSWORD |
| API密钥 | 环境变量 | - |

## 3. 环境变量配置

在 `~/.bashrc` 中添加：

```bash
# 服务器密码
export XIAOAI_PASSWORD="123457"
export ISTORE_PASSWORD="Sy135790@"

# API密钥
export DOUBAO_API_KEY="your-api-key"
```

## 4. GitHub 上传检查清单

- [ ] 检查是否有密码/密钥泄露
- [ ] 检查 .gitignore 是否包含敏感配置
- [ ] 使用 `git diff --staged` 确认提交内容
