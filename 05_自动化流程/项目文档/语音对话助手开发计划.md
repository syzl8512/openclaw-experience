# 语音对话助手 - 项目开发计划

## 1. 项目概述

### 1.1 项目背景

本项目旨在实现一个类"小爱同学"的语音对话系统，用户可以通过语音与 AI 助手进行自然对话。系统采用纯语音交互方式：用户说话 → 语音识别 → AI 处理 → 语音回复，形成完整的对话闭环。

### 1.2 核心功能

| 功能 | 说明 |
|------|------|
| 语音唤醒 | 点击按钮启动语音识别（类似按住说话） |
| 实时语音识别 | 浏览器原生 Web Speech API，支持持续输入 |
| AI 对话处理 | 通过 OpenClaw Gateway 调用 AI Agent 进行语义理解和回复 |
| 语音合成回复 | TTS 将文字转为语音播放，支持多种声音 |
| 对话状态管理 | 支持开始/结束对话，静默超时自动结束 |

### 1.3 目标用户

- 需要 hands-free 操作场景的用户
- 喜欢语音交互的技术爱好者
- 替代传统文字聊天的高效沟通方式

## 3. 功能规格

### 3.1 前端功能

#### 3.1.1 语音输入模块

| 功能 | 描述 | 实现方式 |
|------|------|----------|
| 麦克风权限 | 首次使用时请求麦克风权限 | `navigator.mediaDevices.getUserMedia()` |
| 语音识别启动 | 点击按钮开始语音输入 | `SpeechRecognition.start()` |
| 实时识别显示 | 识别过程中实时显示文字 | `onresult` 回调 |
| 语音输入结束 | 松开按钮或手动结束 | `onend` 回调 |
| 持续对话模式 | 识别完成后自动继续听 | 循环调用 start() |

#### 3.1.2 音频播放模块

| 功能 | 描述 | 实现方式 |
|------|------|----------|
| 音频获取 | 从 TTS 服务获取音频 URL | HTTP 请求 |
| 音频播放 | 播放接收到的语音回复 | `Audio.play()` |
| 播放状态显示 | 显示当前是否在播放 | UI 状态切换 |
| 播放完成检测 | 检测语音播放完毕 | `onended` 事件 |

#### 3.1.3 UI 界面

| 组件 | 描述 |
|------|------|
| 开始按钮 | 大圆形按钮，按住说话 |
| 结束按钮 | 对话中显示，停止语音输入 |
| 对话状态指示 | 正在听 / 正在思考 / 正在说话 |
| 文字显示区 | 显示识别文字和 AI 回复（可选） |
| 历史对话 | 显示最近 N 条对话记录 |

### 3.2 后端功能

#### 3.2.1 WebSocket 服务

| 功能 | 描述 |
|------|------|
| 连接管理 | 处理客户端连接/断开 |
| 消息路由 | 将识别文字转发给 Agent |
| 会话管理 | 维护对话上下文 |
| 心跳检测 | 检测连接存活 |

#### 3.2.2 Agent 对话

| 功能 | 描述 |
|------|------|
| 意图识别 | 理解用户语音输入 |
| 对话管理 | 维护多轮对话上下文 |
| 技能调用 | 根据需求调用各种技能 |
| 格式化输出 | 返回结构化的文字回复 |

#### 3.2.3 TTS 服务

| 功能 | 描述 |
|------|------|
| 文字转语音 | 将回复文字转为语音 |
| 多语音选择 | 支持多种声音（男/女/不同语言） |
| 音频缓存 | 缓存已生成的音频 |
| 流式输出 | 支持流式音频（可选） |

## 7. 关键技术点

### 7.1 语音识别 (豆包 ASR - 首选方案)

```javascript
// 前端：录制音频并发送到后端
const audioBlob = await recordAudio(); // 录制音频
// 发送到后端
formData.append('audio', audioBlob, 'recording.webm');
await fetch('/api/asr', { method: 'POST', body: formData });

// 后端：调用豆包 ASR API
const response = await fetch('https://openspeech.bytedance.com/api/v2/asr', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${DOUBAO_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    app: { appid: APP_ID, token: ACCESS_TOKEN, cluster: 'volcengine_streaming_common' },
    user: { uid: USER_ID },
    audio: {
      format: 'webm',
      rate: 16000,
      bits: 16,
      channel: 1,
      codec: 'raw',
      url: audioUrl
    },
    request: { requestid: uuid(), sequence: 1 }
  })
});
const result = await response.json();
const text = result.result.text; // 识别出的文字
```

### 7.2 备选方案：Web Speech API (浏览器原生)

```javascript
const recognition = new webkitSpeechRecognition();
recognition.continuous = true;
recognition.interimResults = true;

recognition.onresult = (event) => {
  const transcript = event.results[event.results.length - 1][0].transcript;
  // 发送到 WebSocket
  ws.send(JSON.stringify({ type: 'voice_input', text: transcript }));
};
```

### 7.3 TTS 调用 (edge-tts)

```javascript
// 后端 Node.js
const { EdgeTTS } = require('edge-tts');
const tts = new EdgeTTS('zh-CN-XiaoxiaoNeural');
await tts.synthesize('你好，我是 mini');
// 返回音频 Buffer
```

### 7.3 WebSocket 心跳

```javascript
// 客户端
setInterval(() => {
  ws.send(JSON.stringify({ type: 'ping' }));
}, 30000);

// 服务端
ws.on('ping', () => {
  ws.send(JSON.stringify({ type: 'pong' }));
});
```

## 8. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 浏览器不支持 Web Speech API | 部分用户无法使用 | 提示使用 Chrome/Edge |
| 语音识别不准 | 对话体验差 | 显示识别结果供用户确认 |
| TTS 延迟高 | 对话不流畅 | 音频缓存 + 流式输出 |
| 网络不稳定 | 语音对话中断 | 离线提示 + 重连机制 |
